# Система одобрения пользователей (User Approval System)

## Описание

Функция одобрения пользователей позволяет администраторам системы контролировать доступ новых пользователей. При регистрации новый пользователь получает статус "не одобрен" (is_approved=False), и администратор должен вручную одобрить учетную запись.

## Компоненты

### 1. Модель UserProfile
Расширяет функциональность встроенной модели User:

```python
class UserProfile(models.Model):
    user = models.OneToOneField(User)
    is_approved = models.BooleanField(default=False)  # Статус одобрения
    approved_by = models.ForeignKey(User)              # Кто одобрил
    approved_at = models.DateTimeField()               # Когда одобрили
    department = models.CharField()                    # Отдел пользователя
    phone = models.CharField()                         # Телефон
```

**Сигналы:**
- При создании пользователя автоматически создается профиль с is_approved=False
- При сохранении пользователя профиль синхронизируется

### 2. Представления (Views)

#### user_approval_list
- **URL:** `/admin/users/`
- **Доступ:** Только администраторы
- **Функция:** Отображает список пользователей, ожидающих одобрения и одобренных пользователей

#### approve_user
- **URL:** `/admin/users/<user_id>/approve/`
- **Доступ:** Только администраторы
- **Функция:** Одобрение пользователя

#### reject_user
- **URL:** `/admin/users/<user_id>/reject/`
- **Доступ:** Только администраторы
- **Функция:** Отклонение пользователя (деактивация учетной записи)

#### revoke_approval
- **URL:** `/admin/users/<user_id>/revoke/`
- **Доступ:** Только администраторы
- **Функция:** Отозвание одобрения (пользователь остается в системе)

### 3. Формы

- **UserProfileForm:** Для редактирования профиля пользователя
- **UserApprovalForm:** Для одобрения (с опциональным комментарием)
- **UserRejectionForm:** Для отклонения (обязательная причина)

### 4. Шаблоны

- `user_approval_list.html` - Список управления пользователями
- `approve_user.html` - Форма одобрения
- `reject_user.html` - Форма отклонения
- `revoke_approval.html` - Форма отозвания одобрения

### 5. Админ-панель

Добавлен UserProfileAdmin для управления профилями в Django admin:
- Список отобранных пользователей
- Фильтрация по статусу одобрения
- Поиск по имени пользователя или отделу

## Использование

### Для администраторов:

1. Перейти на `/admin/users/` для просмотра списка
2. Выбрать действие:
   - **Одобрить** - одобрить пользователя для доступа
   - **Отклонить** - деактивировать учетную запись
   - **Отозвать** - отозвать ранее выданное одобрение

### Для новых пользователей:

1. Зарегистрироваться в системе
2. Получить статус "ожидает одобрения"
3. Дождаться одобрения администратора
4. Получить полный доступ к системе

## Возможные расширения

- Email-уведомления при одобрении/отклонении
- История одобрения в модели
- Срок действия одобрения
- Требование переподтверждения через N дней
- Импорт пользователей из LDAP/Active Directory с автоматическим одобрением
- Интеграция с системой ролей для дифференцированного доступа
