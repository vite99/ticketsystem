# Функция назначения рабочих мест

## Описание

Добавлена функция для указания конкретного рабочего места (компьютера/рабочей станции), к которому относится проблема. Это особенно полезно в кабинетах с несколькими компьютерами, когда нужно точно указать, какой именно компьютер требует внимания.

## Новые возможности

### 1. Модель Workstation (Рабочее место)
Создана новая модель для управления рабочими местами:

```python
class Workstation(models.Model):
    room = CharField(max_length=100)          # Номер кабинета или офиса
    number = CharField(max_length=50)         # Номер/описание компьютера
    location = CharField(max_length=255)      # Дополнительное местоположение
```

**Уникальность**: Каждое рабочее место уникально в рамках кабинета (комбинация room + number)

**Примеры**:
- Кабинет 101 (IT кабинет) - ПК-1 (Левый угол, у окна)
- Кабинет 101 (IT кабинет) - ПК-2 (Центр, основной стол)
- Кабинет 102 (Офис) - ПК-1 (Левый край)

### 2. Обновлена модель Ticket
Добавлено поле:
```python
workstation = ForeignKey(Workstation, on_delete=SET_NULL, null=True, blank=True)
```

### 3. Формы
Обновлены формы для создания/редактирования тикетов:
- **TicketForm** (для администраторов): включает поле выбора рабочего места
- **TicketFormUser** (для обычных пользователей): включает поле выбора рабочего места

### 4. Интерфейс админ-панели
В админ-панели (django admin) добавлены:

- Фильтрация по кабинетам
- Поиск по номеру и описанию

### 5. Шаблоны

## Использование

### Для администраторов
1. Перейдите в админ-панель: `/admin/`
2. Найдите раздел "Рабочее место"
3. Добавьте новые рабочие места для вашего учреждения
4. При создании/редактировании тикета выберите соответствующее рабочее место

### Для пользователей
1. При создании тикета выберите рабочее место, к которому относится проблема
2. Система автоматически привяжет информацию о рабочем месте к тикету
3. Администратор сможет увидеть точное местоположение проблемы

## Технические подробности

### База данных
Созданы две новые таблицы:
- `tickets_workstation` - таблица рабочих мест
- Обновлена `tickets_ticket` с добавлением поля `workstation_id` (FK)

### Ограничения
- Рабочее место не может быть удалено, если на него ссылаются тикеты (защита целостности данных)
- Комбинация (room, number) должна быть уникальной для предотвращения дубликатов
