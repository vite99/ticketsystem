## 📋 Модели данных

### Модель Workstation (Рабочее место)

```
Workstation
├─ room: CharField(max_length=100)
│  └─ Пример: "101 (IT кабинет)"
├─ number: CharField(max_length=50)
│  └─ Пример: "ПК-1"
├─ location: CharField(max_length=255)
│  └─ Пример: "Левый угол, у окна"
└─ Meta:
   ├─ unique_together: [('room', 'number')]
   └─ ordering: ['room', 'number']
```

### Модель Ticket (обновлена)

```
Ticket
├─ ... (существующие поля)
└─ workstation: ForeignKey(Workstation)
   ├─ null=True
   ├─ blank=True
   ├─ on_delete=SET_NULL
   └─ related_name='tickets'
```

## 🔄 Процесс создания тикета

```
1. Пользователь открывает форму создания
           ↓
2. Заполняет заголовок и описание
           ↓
3. Выбирает приоритет и рабочее место ◄── НОВОЕ
           ↓
4. Нажимает "Создать"
           ↓
5. Django валидирует форму
           ↓
6. Ticket.objects.create(..., workstation=selected_workstation)
           ↓
7. Тикет сохраняется в БД с привязкой к рабочему месту
           ↓
8. Пользователь видит созданный тикет
           ↓
9. Администратор видит информацию о рабочем месте
```

## 🚀 Ключевые особенности

✅ **Безопасность**: Уникальность на уровне БД (unique_together)
✅ **Целостность**: SET_NULL при удалении рабочего места
✅ **Удобство**: Выпадающее меню с готовыми вариантами
✅ **Гибкость**: Опциональное поле (можно не заполнять)
✅ **Локализация**: Все на русском языке
✅ **Масштабируемость**: Легко добавлять новые рабочие места
✅ **Документация**: Подробные инструкции для всех
