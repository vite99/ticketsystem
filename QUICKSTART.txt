🐛 РЕШЕНИЕ ПРОБЛЕМ
──────────────────

Ошибка: "No module named 'rest_framework'"
  Решение: pip install -r requirements.txt

Ошибка: "ModuleNotFoundError: No module named 'django'"
  Решение: source venv/bin/activate && pip install -r requirements.txt

Ошибка: "Database is locked"
  Решение: Закрыть все процессы Django и удалить db.sqlite3, заново запустить migrate

Ошибка: "port 8000 already in use"
  Решение: python manage.py runserver 8001 (использовать другой порт)

Ошибка: "CSRF verification failed"
  Решение: Убедиться что {% csrf_token %} в форме

🗄️ БАЗА ДАННЫХ
────────────────

Текущая БД: SQLite (db.sqlite3)

Инициализировать БД с нуля:
  python manage.py flush
  python manage.py migrate
  python init_db.py

Резервная копия (SQLite):
  cp db.sqlite3 db.sqlite3.backup

Экспорт данных:
  python manage.py dumpdata > data.json

Импорт данных:
  python manage.py loaddata data.json

📧 EMAIL / SMTP
────────────────

Для отправки писем, установите в .env:

  EMAIL_BACKEND=django.core.mail.backends.smtp.EmailBackend
  EMAIL_HOST=smtp.gmail.com
  EMAIL_PORT=587
  EMAIL_USE_TLS=True
  EMAIL_HOST_USER=your-email@gmail.com
  EMAIL_HOST_PASSWORD=app_password_here
  DEFAULT_FROM_EMAIL=noreply@ticketsystem.local

Примечание для Gmail:
  1. Включить двухфакторную аутентификацию
  2. Создать App Password (приложение)
  3. Использовать эту пароль в EMAIL_HOST_PASSWORD
