Q: Как запустить на Windows?
A: Двойной клик на setup.bat или выполнить команды в cmd

Q: Как использовать на Linux?
A: bash setup.sh или выполнить команды в терминале

Q: Как развернуть на production?
A: Прочитайте DEPLOYMENT.md для инструкций по Heroku, Docker или VPS

Q: Как изменить базу данных?
A: Измените DATABASE_URL в settings.py на PostgreSQL или MySQL

Q: Как добавить SMTP для писем?
A: Установите параметры EMAIL_* в settings.py или .env файле

Q: Как расширить функционал?
A: Добавьте новые модели, views, и шаблоны следуя существующему паттерну

Q: Как запустить тесты?
A: python manage.py test tickets

Q: Где найти логи ошибок?
A: Проверьте terminal output или просмотрите django.log если включено логирование

───────────────────────────────────────────────────────────────────────────────

Если возникли проблемы:

1. Проверьте README.md - там есть раздел "Решение проблем"
2. Посмотрите QUICKSTART.txt для быстрых команд
3. Запустите тесты: python manage.py test tickets

5. Убедитесь что все зависимости установлены
6. Прочитайте комментарии в коде

Общие проблемы:
  • "ModuleNotFoundError: No module named..." → pip install -r requirements.txt
  • "Port already in use" → python manage.py runserver 8001
  • "Database is locked" → Закройте все Django процессы
  • "CSRF verification failed" → Проверьте {% csrf_token %} в форме
