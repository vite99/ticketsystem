# 🎫 Ticket System - Система управления тикетами

Веб-система на Django для управления тикетами (задачами, проблемами, запросами) с полной функциональностью отслеживания, назначения и комментирования.

## 📋 Функциональность

- ✅ **Управление тикетами** - Создание, редактирование, просмотр, удаление
- ✅ **Статусы и приоритеты** - Гибкая система категоризации
- ✅ **Назначение задач** - Распределение тикетов между пользователями
- ✅ **Комментарии** - Система коммуникации с внутренними комментариями
- ✅ **Вложения** - Поддержка загрузки файлов
- ✅ **История изменений** - Отслеживание всех действий
- ✅ **Фильтрация и поиск** - Быстрый поиск по названию и описанию
- ✅ **Панель администратора** - Полное управление через Django admin
- ✅ **Личный кабинет** - Просмотр собственных тикетов и статистика
- ✅ **SMTP интеграция** - Отправка уведомлений по почте
- ✅ **Аутентификация** - Система прав доступа

## 🔧 Установка и запуск

### Требования
- Python 3.8+
- pip
- virtualenv (рекомендуется)

### Шаги установки

1. **Клонирование репозитория**
```bash
cd диплом
```

2. **Создание виртуального окружения** (если не создано)
```bash
python -m venv venv
```

3. **Активация виртуального окружения**

**Windows:**
```bash
venv\Scripts\activate
```

**Linux/Mac:**
```bash
source venv/bin/activate
```

4. **Установка зависимостей**
```bash
pip install -r requirements.txt
```

5. **Применение миграций**
```bash
python manage.py migrate
```

6. **Создание суперпользователя** (для доступа в админку)
```bash
python manage.py createsuperuser
```

7. **Запуск сервера разработки**
```bash
python manage.py runserver
```

Приложение будет доступно по адресу: `http://localhost:8000/`

## 📊 Модели данных

### Priority (Приоритет)
- LOW - Низкий
- MEDIUM - Средний
- HIGH - Высокий
- CRITICAL - Критический

### Status (Статус)
- OPEN - Открыт
- IN_PROGRESS - В работе
- WAITING - Ожидание
- RESOLVED - Решен
- CLOSED - Закрыт
- REOPENED - Переоткрыт

### Ticket (Тикет)
- Основная сущность системы
- Связанные таблицы: Priority, Status, Tag
- Автор, назначенный пользователь
- Описание, временные метки
- Срок выполнения, расчетные часы

### Comment (Комментарий)
- Привязан к Ticket
- Автор (User)
- Флаг внутреннего комментария

### Attachment (Вложение)
- Может быть привязано к Ticket или Comment
- Валидация типов файлов
- Метаданные загрузки

### TicketHistory (История)
- Отслеживает все изменения
- Сохраняет старые и новые значения

## 🔐 Аутентификация и разрешения

- **Логин**: `/login/` - форма входа
- **Личный кабинет**: `/dashboard/` - просмотр своих тикетов
- **Админка**: `/admin/` - полное управление (только для персонала)

Правила доступа:
- Посмотреть тикет - все авторизованные пользователи
- Редактировать тикет - создатель, назначенный пользователь или админ
- Создавать комментарии - все авторизованные пользователи

## 📧 SMTP конфигурация

Для отправки уведомлений по почте, добавьте в `.env`:

```env
EMAIL_BACKEND=django.core.mail.backends.smtp.EmailBackend
EMAIL_HOST=smtp.gmail.com
EMAIL_PORT=587
EMAIL_USE_TLS=True
EMAIL_HOST_USER=ваш_email@gmail.com
EMAIL_HOST_PASSWORD=ваш_пароль_приложения
DEFAULT_FROM_EMAIL=noreply@ticketsystem.local
```

## 📝 API Endpoints (планируется)

- `GET /api/tickets/` - Список тикетов
- `POST /api/tickets/` - Создание тикета
- `GET /api/tickets/<id>/` - Детали тикета
- `PUT /api/tickets/<id>/` - Обновление тикета
- `DELETE /api/tickets/<id>/` - Удаление тикета
- `GET /api/tickets/<id>/comments/` - Комментарии

## 🌐 Развертывание

### Docker

Создан Dockerfile для контейнеризации приложения:

```bash
docker build -t ticket-system .
docker run -p 8000:8000 ticket-system
```

### Production

Для развертывания на production сервере:

1. Измените `DEBUG = False` в `settings.py`
2. Установите `SECRET_KEY` через переменные окружения
3. Настройте базу данных PostgreSQL
4. Используйте gunicorn/uwsgi + nginx
5. Включите HTTPS

# Импорт

## 🐛 Известные проблемы и ограничения

- Система работает на SQLite в dev режиме (для production используйте PostgreSQL)
- Максимальный размер загружаемого файла - 50MB (настраивается в settings.py)
- Система не поддерживает кириллицу в URL (планируется исправить)
